## Context

卡拉ok工作台（krok-helper）目前已完成 6 步工作流中的 5 步（视频下载 / 波形对齐 / 歌词检索 / 歌词打轴 SUG / Hi-Res 混流），第 5 步"字幕视频渲染"在 `gui_qt.py` 中原本是 `PlaceholderPage`，骨架阶段已替换为本模块的嵌入实例。本模块是发布 1.0 前最后一个、也最复杂的模块，目标是实现 **NicoKaraMaker3 等效功能**：把带逐字时间戳的歌词，渲染为带卡拉ok高亮动画字幕的视频文件（叠加在 MV 或纯背景上）。

模块必须支持双模式：

开发策略：在 `feat/subtitle-render` 长线开发，期间 `main` 上的 bugfix 频繁 merge 进开发分支（`git merge main`），最终一次性 merge 回 main。**不要把零散 bugfix 直接提交到开发分支**——bugfix 走 main。

复刻 SUG（lyrics_timing 子模块）的双模式骨架：`MainWindow(embedded: bool)` + `for_embedding()` 工厂方法 + `KrokHelperSubtitleRenderSettingsBridge` 设置桥。

## B. 双模式数据流

### Standalone
- **入口**：`python -m krok_helper.subtitle_render` / 菜单"文件 → 新建"
- **加载**：三个独立选择按钮——字幕（`.lrc`）/ 背景视频 / 音频
- **存储**：项目 → `.yurika.json`；全局首选项 → `AppSettings.subtitle_render` dict
- **装饰**：完整标题栏 + 菜单栏（文件 / 编辑 / 视图 / 帮助）+ 状态栏

### Embedded
- **构造**：`SubtitleRenderWindow.for_embedding(parent, settings_provider, workflow_context)`
- **上游数据**（自动获得，左侧素材区只读）：
  - 字幕：`context.subtitle_lrc_path`（SUG `NicokaraExporter` 落盘的 Nicokara 逐字 LRC）。SUG 完成第 4 步时自动调用 `NicokaraExporter` 落盘到工作流目录约定位置（如 `output/{slug}/subtitle.lrc`）
  - 背景视频：`context.aligned_video_path`（第 2 步对齐后的 1080p60 mp4），**默认作为背景层**
  - 音轨：与背景视频同源（mux）或独立流
  - 偏移：`context.alignment_offset_ms`
- **下游交付**：写 `context.subtitle_video_path`，发 `workflow_step_completed` 信号
- **去装饰**：参考 SUG 的 `_remove_embedded_title_bar()` / `_apply_embedded_ui_policy()` 模式（位于 `krok_helper/lyrics_timing/src/strange_uta_game/frontend/main_window.py:212` / `:236`）

### 设置桥
- 骨架已实现 `KrokHelperSubtitleRenderSettingsBridge`（`krok_helper/subtitle_render/settings_bridge.py`），仿照 SUG 的 `KrokHelperSettingsBridge`（`gui_qt.py:182-244`）
- 暴露字段示例：`default_font_family` / `default_resolution` / `default_fps` / `prefer_hwenc` / `output_dir` / `recent_style_presets`——存进 `AppSettings.subtitle_render` dict 的对应 key

## D. 数据模型（`krok_helper/subtitle_render/models.py`）

骨架阶段只有占位 dataclass，需按下方结构补全字段：

```
RenderProject (dataclass, JSON 序列化)
  ├ subtitle_source: SubtitleSource
  │    ├ path: str                      # 必须是 Nicokara 逐字 LRC (.lrc)
  │    └ singer_filter: list[int] | None  # 按演唱者过滤显示（None=全部）
  ├ background: Background
  │    ├ kind: Literal["solid","image","video","loop_video"]
  │    ├ color: str
  │    ├ path: str | None
  │    └ video_offset_ms: int
  ├ audio_path: str | None
  ├ global_style: Style
  ├ singer_styles: dict[int, Style]
  ├ line_style_overrides: dict[int, Style]   # P2
  ├ title_overlay: TitleOverlay | None
  ├ output: OutputConfig
  └ schema_version: int = 1

Style: font_family / font_size_px / weight / italic / vertical / fill_color
       / fill_color2 / gradient_angle_deg / base_color / stroke_color
       / stroke_width_px / shadow_color / shadow_offset / glow_radius_px (P2)
       / ruby_size_ratio / ruby_color / ruby_vertical_offset_px
       / entry_anim / entry_lead_ms / exit_anim / exit_fade_ms / line_y_position

OutputConfig: width / height / fps / video_codec / encoder / crf / preset
              / pix_fmt / audio_codec / audio_bitrate_kbps / output_path
```

- 项目文件后缀：`.yurika.json`
- 样式预设后缀：`.krstyle.json`
- `schema_version` 字段保留升级空间

## E. 渲染引擎技术选型

**已选定：QPainter 离屏 + ffmpeg rawvideo pipe**（不要改方案，已与用户达成一致）

- 流程：枚举每帧 t → `QPainter` 画到 `QImage(Format_RGBA8888)` → bytes → `subprocess.Popen` 写入 ffmpeg stdin (`-f rawvideo -pix_fmt rgba`) → ffmpeg 同时读 `-i background` 和 `-i audio` 做合成与编码
- **关键优势**：
  1. 卡拉ok填充逐像素精确，渐变/发光/动画完全自由
  2. **预览与渲染共用 QPainter 绘制函数**，所见即所得
  3. 直接复用 `krok_helper/ffmpeg.py` 的 `run_command` / `find_tool` / `probe_media`
  4. 硬编探测沿用 `audio_alignment.py::export_aligned_video_v2`（参考其分支判断写法）
- **缓解 CPU 占用**：多线程绘帧（producer N×QPainter → 队列 → 单 consumer 喂 ffmpeg），目标 1080p60 实时倍率 ≥ 0.5×

**预览策略**：同引擎降分辨率（`QImage(960, 540)`），定位用 `QMediaPlayer.setPosition()` 取静态背景帧，叠加 QPainter 字幕。保证"预览=输出"。

**否决方案**（避免重新讨论）：
- ❌ 生成 `.ass` + libass burn-in：渐变 / 发光 / 复杂动画表达受限；预览路径割裂。可作为 P2 "极速导出"按钮，但不是主路径。
- ❌ Pillow / Skia / Web 离屏：与现有 PyQt 基建脱节、PyInstaller 打包冲突。

## H. 已确认的产品决策（不要重新讨论）

1. **嵌入模式下字幕传递**：**单一文件路径**——SUG 第 4 步完成时调用 `NicokaraExporter` 把 Project 落盘成 Nicokara 逐字 LRC，第 5 步通过 `WorkflowContext.subtitle_lrc_path` 读文件。
   - 不再走内存对象直传：实现简单、调试方便、和 standalone 模式共用同一路径
   - 模块内部只需要一个入口 `load_from_lrc(path)`
   - 若将来发现 IO 成为瓶颈再考虑内存直传补丁
2. **嵌入模式背景默认行为**：**默认叠在第 2 步对齐后的 MV 上**。
   - 素材区"背景"默认值 = `Background(kind="video", path=context.aligned_video_path)`
   - 用户仍可在属性面板切换为纯色 / 图片 / 循环视频
3. **垂直書き（縦書き）**：**推迟到 P1（B9）**，MVP 仅支持横书き。
   - 影响 Style 字段：`vertical` 标志位 MVP 保留但只允许 `False`；P1 阶段实装字形旋转 / 标点替换 / ruby 横转纵
4. **样式预设包**：**MVP 只给空白默认**。
   - `style_presets.py` MVP 阶段实现保存 / 加载 `.krstyle.json` 的能力（B5）但不附带任何内置预设包
   - 预置预设包后续作为独立工作单独排期，可能由非工程师贡献
